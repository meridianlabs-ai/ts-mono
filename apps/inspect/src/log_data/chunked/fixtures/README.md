# Chunked-format test fixtures

## `skeleton/`

The shared skeleton fixture suite. **Source of truth:
`tests/log/test_skeleton/` in the inspect_ai repo** — the same JSON files
run against the Python producer (`inspect_ai/log/_skeleton.py`) and the TS
twin (`../skeleton.ts`). Never edit here; re-copy from inspect_ai when the
suite changes:

```sh
cp <inspect_ai>/tests/log/test_skeleton/*.json skeleton/
```

## `logs/`

Small real `.eval` logs (`original/`) and their chunked conversions
(`chunked/`), generated by the inspect_ai converter with `chunk_size=3` so
every sequence is multi-chunk. Regenerate with:

```python
from inspect_ai.log._recorders.chunked import convert_eval_logs_to_chunked

convert_eval_logs_to_chunked(original_log, "chunked/", overwrite=True, chunk_size=3)
```

Sources (under the inspect_ai repo's `tests/`):

- `2024-11-05…input-task….eval` — span-rich modern log (the parity-oracle
  per-span sweep log)
- `2025-02-11…popularity….eval` — legacy step-only log
- `log_streaming.eval`, `log_read_sample.eval` — small multi-sample logs
- `ais-decoder.eval` — agent log with checkpoint spans exited in another
  context (unbalanced span order; from the inspect_ai repo's `logs/old/`) —
  the reproducer for the main-view outline twin (`mainViewOutline.test.ts`)
